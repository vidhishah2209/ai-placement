# AI Placement Cell — Comprehensive Project Overview

This document provides a highly detailed breakdown of everything built in the **AI Placement Cell** project: a production-ready, LangGraph-powered multi-agent system designed to analyze, score, and improve tech resumes for Indian campus placements.

---

## 🏗️ 1. Architecture & Technology Stack

### Backend
- **Framework**: FastAPI (Python)
- **Database**: PostgreSQL (asyncpg) with SQLAlchemy ORM (currently defaulting to local SQLite for rapid dev)
- **AI Orchestration**: LangGraph (`StateGraph`)
- **LLM Provider**: Groq API (`llama-3.3-70b-versatile`) with a fallback mechanism to Google Gemini API. LangChain is used for interactive chat.
- **Tools**: PyMuPDF & PyPDF2 (PDF Parsing), `python-dotenv` for configuration management.

### Frontend
- **Framework**: React.js 18 with Vite
- **Language**: TypeScript (`.tsx` / `.ts`)
- **Styling**: Tailwind CSS & Framer Motion for smooth animations and a premium glassmorphic dark-theme design.
- **UI Components**: custom `shadcn/ui` logic and `lucide-react` for iconography.
- **PDF Generation**: `html2pdf.js` for client-side printable resume exporting.

---

## 🧠 2. The 8-Agent LangGraph Pipeline

The core of the backend is an 8-stage AI pipeline. When a user uploads a PDF, the raw text is extracted through PyPDF2 and passed through these specialized agents sequentially via a shared `GraphState`:

1. **`agent_1_parser.py`**: Extracts structured JSON data (Name, Education, Skills, Projects, Experience) from raw PDF text.
2. **`agent_2_ats.py`**: Acts as a strict Indian IT recruiter. Scores the resume (0-100) across categories formatting, keywords, metrics, and coherence. Evaluates and identifies strong matches.
3. **`agent_3_skill_gap.py`**: Identifies Critical Missing Skills and Recommended Skills by comparing the *entire* parsed resume and raw text against the user's Target Role. It guarantees no false positives (if a skill is mentioned anywhere in a project, it's considered a strength).
4. **`agent_4_rewriter.py`**: Rewrites weak bullet points using the STAR method, adds stronger action verbs, truncates fluff to max 3 bullets per project, and ensures the content fits cleanly onto one printable page.
5. **`agent_5_supervisor.py`**: A quality-control agent that checks the rewriter's output for AI hallucinations or fabricated skills. If it fails, it routes the graph back to `agent_4` for a rewrite.
6. **`agent_6_db_saver.py`**: Saves the analysis metadata and final structured JSON to the local database.
7. **`agent_7_interview.py`**: Generates 15+ highly specific interview questions grouped into specific technical, HR, and project depth categories. Provides context, model answers, and follow-ups.
8. **`agent_8_confidence.py`**: Calculates a final "Placement Confidence Score" percentage and tags readiness across Service-based, Mid-Tier Product, and Top-Tier Product companies.

## 🚀 3. Core Features Implemented

The application currently supports the following primary workflows and features:

1. **Resume Processing with Interactive Loading UX**: During the 60-second pipeline execution, users are shown an interactive MCQ Placement Quiz that tests their recruitment knowledge and provides immediate explanations to pass the time.
2. **Tabbed Interview Preparation**: Interview questions generated by the AI are sorted into 3 easy-to-digest tabs (Technical, HR, Project) to avoid overwhelming the candidate.
3. **One-Click PDF Export**: The improved resume JSON is rendered into a hidden, clean, ATS-compliant, 1-page HTML template, which is instantly exported to a styling-preserved `.pdf` via `html2pdf.js`.
4. **Interactive Placement Chatbot**: A floating glassmorphic chatbot widget connected to a LangChain + Groq backend endpoint (`/chat/`) that allows students to ask direct, ad-hoc questions about their career or the AI's feedback.
5. **Interactive Dashboard**: A multi-panel React UI enabling users to toggle between ATS Scores, Skill Gaps, their Improved Resume, Interview Prep, and their overall Confidence Tier.

## 📂 4. Project Directory Breakdown

### `e:\AI_Project\backend\` (FastAPI)
- **`app/main.py`**: Exposes `/analyze-resume/`, `/create-user`, `/login`, and `/chat/`. Sets CORS to allow frontend connections on 5173.
- **`app/agents/pipeline.py`**: The LangGraph StateGraph builder that wires the agents together.
- **`app/agents/agent_*.py`**: The LLM prompt logic and JSON parsers for the 8 stages.
- **`app/agents/gemini_client.py`**: Unified LLM request client.
- **`app/database.py` & `app/models.py`**: SQLAlchemy config and table definitions (`User`, `Resume`).
- **`app/routes/chat.py`**: The endpoint powering the floating frontend placement chatbot.

### `e:\AI_Project\frontend\` (React + Vite)
- **`src/components/Dashboard.tsx`**: The main interface layout after login. Handles navigation state between the 5 analysis sections.
- **`src/components/panels/`**:
  - `UploadPanel.tsx`: File dropzone, target role selection, and dynamic tip carousel loading state.
  - `ATSPanel.tsx` & `SkillsPanel.tsx`: Visualizes strengths, gaps, and priority action items.
  - `ResumePanel.tsx`: Displays the rewritten resume timeline. Contains the hidden print layout and PDF export logic.
  - `InterviewPanel.tsx`: Displays the categorized, tabbed accordion of Q&A.
- **`src/components/Chatbot.tsx`**: The interactive, auto-scrolling assistant widget.
- **`src/types/index.ts`**: Strict TypeScript interfaces mapping directly to the backend JSON outputs.

## ⚙️ 5. Exact Steps to Run the Project Locally

You must start the backend Server and the frontend UI in two separate terminal windows.

### Terminal 1: Starting the Backend Server
```powershell
# 1. Navigate to the backend directory
cd e:\AI_Project\backend

# 2. Activate the Python Virtual Environment
# Windows:
venv\Scripts\activate
# (Mac/Linux: source venv/bin/activate)

# 3. Ensure your .env file is present with valid API keys:
# GROQ_API_KEY="..."
# GEMINI_API_KEY="..."

# 4. Start the FastAPI server on port 8000
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
> *Look for "✅ Tables created successfully!" and "Application startup complete" in the logs.*

### Terminal 2: Starting the Frontend UI
```powershell
# 1. Navigate to the frontend directory
cd e:\AI_Project\frontend

# 2. Start the Vite React development server
npm run dev
```
> *The terminal will show `Local: http://localhost:5173/`. Open this link in your browser.*

### How to test:
1. Open `http://localhost:5173` in your browser.
2. Enter any name/email on the Auth Screen to "login" (local testing ignores auth passwords).
3. Upload a sample Resume (PDF).
4. Select a Target Role and click "Analyze Resume". Wait for the loading carousel to finish.
5. Explore the various tabs (ATS, Skills, Interview Tabs, PDF Export, and click the Chatbot icon in the bottom right corner!).
